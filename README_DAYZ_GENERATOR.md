# DayZ MODtypes Generator с GUI

Утилита для автоматического создания модульных файлов типов предметов для DayZ Standalone сервера. Работает в паре с `cfgeconomycore.xml` для разделения ванильных и модовых предметов.

## 📋 Описание

Эта утилита позволяет:
- Создавать отдельные файлы `_types.xml` для каждого мода
- Автоматически обновлять `cfgeconomycore.xml` для подключения новых файлов типов
- Управлять предметами через удобный графический интерфейс
- Импортировать/экспортировать конфигурации предметов в JSON
- Автоматически определять категории и теги предметов по имени

## 🔧 Требования

- Python 3.8+
- tkinter (обычно входит в состав Python)
- Для Linux: `sudo apt-get install python3-tk`

## 🚀 Запуск

### Графический интерфейс (рекомендуется)

```bash
python3 dayz_modtypes_gui.py
```

## 📖 Как использовать

### 1. Выбор папки миссии
Нажмите кнопку **"Обзор..."** и выберите папку вашей миссии (например, `mpmissions/dayzOffline.chernarusplus`)

### 2. Настройка мода
- **Название мода**: Уникальное имя вашего мода (например, `BaseBuildingPlus`, `CodeLock`)
- **Папка типов**: Папка где будут храниться файлы типов (по умолчанию `CustomTypes`)

### 3. Добавление предметов

#### Через интерфейс:
1. Нажмите **"➕ Добавить"**
2. Введите название предмета (точно как в configs.cpp мода)
3. Настройте параметры спавна:
   - **Nominal**: Целевое количество предмета на карте
   - **Min**: Минимальное количество перед рестартом экономики
   - **Quantmin/Quantmax**: Диапазон количества в стаке
   - **Max**: Максимальное количество на карте
   - **Restock min/max**: Время пополнения запасов (секунды)
4. Выберите категорию или нажмите **"🎯 Авто"** для автозаполнения
5. Выберите тег размещения (floor, shelves, hangers, ground)

#### Импорт из JSON:
1. Нажмите **"📥 Импорт JSON"**
2. Выберите файл с конфигурацией предметов
3. Предметы будут добавлены в список

### 4. Генерация файлов

#### Вариант A: По отдельности
- **"🔄 Обновить cfgeconomycore.xml"** - добавляет ссылку на файл типов мода
- **"💾 Создать MODtypes.xml"** - генерирует файл с типами предметов

#### Вариант B: Всё сразу
- **"🚀 Выполнить всё"** - выполняет оба действия последовательно

## 📁 Структура файлов

После генерации создаётся следующая структура:

```
mpmissions/ваша.миссия/
├── cfgeconomycore.xml          # Главный конфиг экономики
├── CustomTypes/                 # Папка с типами модов
│   ├── MyMod_types.xml         # Типы предметов вашего мода
│   └── AnotherMod_types.xml    # Типы других модов
└── ...
```

## 📄 Пример cfgeconomycore.xml

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<economycore>
    <ce folder="CustomTypes">
        <file name="MyMod_types.xml" type="types" />
        <file name="BaseBuildingPlus_types.xml" type="types" />
    </ce>
</economycore>
```

## 📝 Формат JSON для импорта

```json
[
  {
    "name": "MyMod_Rifle_AKM",
    "nominal": 10,
    "min": 4,
    "quantmin": 0,
    "quantmax": 100,
    "max": 20,
    "restock_min": 1800,
    "restock_max": 3600,
    "category_main": "weapons",
    "category_sub": "rifles",
    "usage_flags": ["military"],
    "tag": "shelves"
  }
]
```

## 🎯 Категории предметов

### Основные категории:
- `weapons` - Оружие
- `food` - Еда и напитки
- `clothes` - Одежда
- `tools` - Инструменты
- `medic` - Медицина
- `containers` - Контейнеры
- `vehiclesparts` - Части транспорта
- `misc` - Разное

### Теги размещения:
- `floor` - На пол
- `shelves` - На полки
- `hangers` - На вешалки
- `ground` - На землю

## ⚠️ Важные замечания

1. **Переопределение**: Если тип предмета совпадает с ванильным, ваш тип полностью заменит стандартный
2. **Полнота данных**: При переопределении необходимо указывать все параметры
3. **Уникальные имена**: Предметы из модов должны иметь уникальные имена классов
4. **Перезагрузка сервера**: После изменений требуется перезапуск сервера

## 🔧 Автоматическое определение

Кнопка **"✨ Авто-заполнить"** автоматически определяет:
- Категорию предмета по ключевым словам в названии
- Тег размещения по типу предмета

Примеры распознавания:
- `rifle`, `pistol`, `ammo` → weapons/rifles, weapons/pistols, weapons/ammo
- `can`, `food`, `bread` → food/cookedfood
- `shirt`, `pants`, `jacket` → clothes/tops
- `axe`, `shovel`, `hammer` → tools/handtools
- `bandage`, `pills`, `medicine` → medic

## 🐛 Решение проблем

### Ошибка "ImportError: libtk8.6.so"
Установите tkinter:
```bash
# Ubuntu/Debian
sudo apt-get install python3-tk

# CentOS/RHEL
sudo yum install python3-tkinter

# Windows/Mac: обычно входит в состав Python
```

### Файл cfgeconomycore.xml не создаётся
Проверьте права доступа к папке миссии. Утилита должна иметь права на запись.

### Предметы не появляются в игре
1. Проверьте правильность названия класса предмета (должно совпадать с configs.cpp)
2. Убедитесь что мод загружен на сервере
3. Проверьте логи сервера на ошибки экономики
4. Перезапустите сервер после изменений
